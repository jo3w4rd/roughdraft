    Struct XRDisplaySubsystem.XRRenderPass

    Contains configuration parameters about which view into the Scene the renderer should rasterize, and a render target (which can be a texture array) for the result of the rasterization.

    Namespace: UnityEngine.Experimental.XR
    Syntax
    public struct XRRenderPass
    Remarks

    An XRRenderPass can contain more than one XRRenderParameter (viewpoints that the render pipeline renders to the output texture as either different viewports or texture array slices). The render pipeline must query each child XRRenderParameter via Experimental.XR.XRDisplaySubsystem.XRRenderPass.GetRenderParameter|GetRenderParameter. The most optimal way to implement an XRRenderPass is to cull first, and then submit draw calls once for the resulting objects. You can also use techniques such as instanced rendering to optimize XRRenderPasses that contain more than one XRRenderParameter.

    XRRenderPass is typically consumed by a scriptable rendering pipeline.

    Remarks

    Multiple [XRRenderPass|render passes](xref:UnityEngine.Experimental.XR.XRRenderPass|render passes) can share the same index. This means that the renderer only needs to cull once, and can reuse the result of the culling for all render passes that use the same index.
