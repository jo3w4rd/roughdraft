    Class NavMesh

    Singleton class to access the baked NavMesh.

    Inheritance
    Object
    NavMesh
    Namespace: UnityEngine.AI
    Syntax
    public static class NavMesh
    Remarks

    The NavMesh is a class can be used to do spatial queries, like pathfinding and walkability tests, set the pathfinding cost for specific area types, and to tweak global behavior of pathfinding and avoidance.

    In order to use the spatial queries, you need to first bake NavMesh for your Scene.

    See also:

    • [[wiki:nav-BuildingNavMesh|Building a NavMesh]] – for more information on how to setup and bake NavMesh

    • [[wiki:nav-AreasAndCosts|Areas and Costs]] – to learn how to use different Area types.

    • NavMeshAgent – to learn how to control and move NavMesh Agents.

    • NavMeshObstacle – to learn how to control NavMesh Obstacles using scripting.

    • OffMeshLink – to learn how to control Off-Mesh Links using scripting.

    Fields

    AllAreas

    Area mask constant that includes all NavMesh areas.

    Declaration
    public const int AllAreas = -1
    Field Value
    Type Description
    Int32
    Remarks

    The mask can be used in query functions, such as NavMesh.Raycast, to indicate that all NavMesh area types are accepted.

    Examples
    
// TargetReachable
using UnityEngine;
using UnityEngine.AI;

public class TargetReachable : MonoBehaviour
{
    public Transform target;
    private NavMeshHit hit;
    private bool blocked = false;

    void Update()
    {
        // Allow pass through all area types when testing if the target position
        // is reachable from the transform location.
        blocked = NavMesh.Raycast(transform.position, target.position, out hit, NavMesh.AllAreas);
        Debug.DrawLine(transform.position, target.position, blocked ? Color.red : Color.green);
        if (blocked)
            Debug.DrawRay(hit.position, Vector3.up, Color.red);
    }
}

    Properties

    avoidancePredictionTime

    Describes how far in the future the agents predict collisions for avoidance.

    Declaration
    public static float avoidancePredictionTime { get; set; }
    Property Value
    Type Description
    Single
    Remarks

    The larger the value, the earlier the agents will start to avoid each other if they are on collision course. The value is measured in seconds. Default value is 2.0, a good range for tuning is between 0.5 and 5.0.

    pathfindingIterationsPerFrame

    The maximum amount of nodes processed each frame in the asynchronous pathfinding process.

    Declaration
    public static int pathfindingIterationsPerFrame { get; set; }
    Property Value
    Type Description
    Int32
    Remarks

    The pathfinder expands only a certain amount of nodes (NavMesh polygons) each frame while doing path finding. This allows the game to run smoothly without hiccups when long paths or large number of request are being processed at the same time, the flip side is that it can take number of frames for a path request to complete.

    The iteration count only affects asyncronous pathfinding which used when setting NavMesh Agent destination using AI.NavMeshAgent.SetDestination or AI.NavMeshAgent.destination.

    Increasing this value will cause the paths to be processed faster, but can cause some hiccups in frame rate. Default value is 100, a good range for tuning is between 50 and 500.
