    Syntax
    public class Component : Object
    Remarks

    Note that your code will never directly create a Component. Instead, you write script code, and attach the script to a GameObject. SA: ScriptableObject as a way to create scripts that do not attach to any GameObject.

    tag

    The tag of this game object.

    Declaration
    public string tag { get; set; }
    Property Value
    Type Description
    String
    Remarks

    A tag can be used to identify a game object. Tags must be declared in the [[wiki:class-TagManager|Tags and Layers manager]] before using them.

    Note: You should not set a tag from the MonoBehaviour.Awake|Awake() or MonoBehaviour.OnValidate|OnValidate() method. This is because the order in which components become awake is not deterministic, and therefore can result in unexpected behaviour such as the tag being overwritten when it is awoken. If you try this, Unity will generate a warning reading "SendMessage cannot be called during Awake, CheckConsistency, or OnValidate".

    Examples
    
using UnityEngine;

public class Example : MonoBehaviour
{
    void Start()
    {
        // Prints the tag that this transform has
        Debug.Log("Transform Tag is: " + gameObject.tag);
    }
}
