    Struct SubMeshDescriptor

    Contains information about a single sub-mesh of a Mesh.

    Namespace: UnityEngine.Rendering
    Syntax
    public struct SubMeshDescriptor
    Remarks

    Simple usage of Mesh scripting API involves using functions like Mesh.triangles, Mesh.vertices and so on.

    For advanced use cases that require maximum performance, you can use the advanced API, which has functions like Mesh.SetSubMesh, Mesh.SetIndexBufferParams, and Mesh.SetIndexBufferData. This advanced API gives access to the underlying mesh data structures that primarily work on raw index buffers, vertex buffers and mesh subset data.

    A single sub-mesh represents part of the Mesh that is using one material. Many Meshes use just one material, but some might use more. Information in a sub-mesh is composed of:

    • ::ref::bounds - bounding box of vertices in local space.

    • ::ref::indexStart - starting point inside the whole Mesh index buffer where the face index data of this subset is found. See Mesh.SetIndexBufferParams and Mesh.SetIndexBufferData.

    • ::ref::indexCount - index count for this sub-mesh. For example, in meshes with triangle topology, each triangle needs three indices.

    • ::ref::topology - topology of this sub-mesh, most often MeshTopology.Triangles.

    • ::ref::baseVertex - offset that is added to each value in the index buffer, to compute the final vertex index.

    • ::ref::firstVertex and ::ref::vertexCount - range of vertices that are referenced by the index buffer of this sub-mesh.

    SA: Mesh.SetSubMesh, Mesh.GetSubMesh, Mesh.SetIndexBufferParams, Mesh.SetIndexBufferData.

    baseVertex

    Offset that is added to each value in the index buffer, to compute the final vertex index.

    Declaration
    public int baseVertex { get; set; }
    Property Value
    Type Description
    Int32
    Remarks

    You can use the Base vertex to achieve meshes that are larger than 65535 vertices while using 16 bit index buffers, as long as each sub-mesh fits within its own 65535 vertex area. For example, if the index buffer values are 10,11,12 and /baseVertex/ is set to 100000, then effectively vertices 100010, 100011 and 100012 will be used for rendering.

    SA: Mesh.SetSubMesh, Mesh.GetSubMesh, Mesh.SetIndexBufferParams, Mesh.SetIndexBufferData.

    This property, together with ::ref::indexStart, determines which part of the whole index buffer is used by a particular sub-mesh.     vertexCount

    Number of vertices used by the index buffer of this sub-mesh.

    Declaration
    public int vertexCount { get; set; }
    Property Value
    Type Description
    Int32
    Remarks

    Together with ::ref::firstVertex, this defines the range of vertices that are possibly used by this sub-mesh. You must set the vertex count so that @@firstVertex+vertexCount@@ encompasses the whole range of vertices referenced by the index buffer. For example, if the index buffer contains vertex indices @@5,6,100@@, then the /firstVertex/ needs to be @@5@@, while /vertexCount/ needs to be @@96@@ (100-5+1).

    These values are important for any CPU-based mesh processing, for example dynamic batching or CPU skinning.
