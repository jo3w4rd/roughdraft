What?

A permalink uses an anchor on index.md to redirect to a location elsewhere in the document.

Why?

This decouples external links on the dashboard from the internal structure of the documentation files and folders.

How?

To add a permanent link to an anchor inside the docs:

  1. Define the anchor in that doc.

  2. Add a mapping element to the permalink object defined at the top of index.md. Such as:

     defaultql : "articles/DashboardConfigurations.html#defaultql",

    a. The key is the permanent anchor. b. the value is the URL relative to index.md.

  3. Add the key as an anchor to index.md. (This is just to prevent doc build errors complaining about missing links in the file.)

     <a name="defaultql" id="#defaultql" class='anchor'></a>

  4. Add a test link to the list above so we can easily make sure all the links work after making a build.
