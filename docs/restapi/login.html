Authorization

Obtain or refresh OAuth 2.0-style access tokens.

Version: 2.0.0

/login

POST

Summary: Requests the login tokens authorizing API operations.

Description: Use your Unity account username and password.

Parameters

Name Located in Description Required Schema
credentials body Login credentials for a Unity account with access to the project. Note, to refresh a login, use the refresh token as the credential rather than the username and password. Yes credentials

Responses

Code Description Schema
200 Login successful. Access tokens are returned in the response body. authorizationTokens
401 Unauthorized. Check that your credentials object is well formed and contains valid credentials.

Models

credentials

Unity account login credentials. (To refresh a login, you can send refreshCredentials instead of user credentials in the request body.)

Name Type Description Required
username string A Unity developer account username. Yes
password string The account password. Yes
grant_type string Always: 'PASSWORD' Yes

refreshCredentials

Request body for obtaining a new access token using a refresh token.

Name Type Description Required
refresh_token string A refresh_token obtained from a user-level login request. Yes
grant_type string Always: 'refresh_token' Yes

authorizationTokens

Tokens obtained by a successful login operation.

Name Type Description Required
access_token string The token authorizing access to the Tutorial Manager service. Expires after expires_in seconds. Use this token in the Authorization header for all Tutorial Manager API operations. No
refresh_token string Use this token to refresh an expired access token without requiring the user's credentials. (Otherwise, login again with the username and password.) No
expires_in integer The time, in seconds, until the access token expires. No
