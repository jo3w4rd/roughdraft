Tutorial Manager REST API

Use the Tutorial Manager REST API to manage your remote tutorial configuration. The REST API duplicates most of the functions of the Tutorial Manager dashboard. Use the REST API to integrate with build processes or other external systems.

The operations you can take using the REST API include:

  • get_tutorials — Get a JSON string containing the tutorial configuration from either the development or release environment.
  • put_tutorials — Replace the current tutorial configuration with a new one. You can only replace the configuration in the development environment. To replace the release configuration, you must first put the new configuration to the development environment and then move the configuration to the release environement with a copy operation.
  • copy — Copy the configuration from one environment to the other, overwriting all information in the destination.

To perform any of these operations, you must include an authorization header that includes an access token obtained using the login operation. Logging in requires the username and password of a Unity account with access to the project.

Note: You must push the initial configuration for a project from the Unity Editor before you can use this REST API.

See Managing Tutorials for more information about manual dashboard operations and features.

Configuration structure

A Tutorial Manager configuration contains all tutorials, tutorial steps, and step text and enabled flags for a project.

The configuration is transmitted as an array of JSON objects, with each item in the array representing a specific tutorial element or list of elements:

{
    "remoteSettings": [{
        "key": "name",
        "type": "data type",
        "value": "stringified list or value"
    }]
}

Objects in the array have the following properties:

Property Purpose
key A string constructed to uniquely identify each part of the configuration. For example, the key, "tutorials", identifies the list of tutorials in the configuration, a key set to the tutorial name identifies the list of steps in that tutorial, etc. Some keys are concatenated from the tutrial and step key names.
type The data type stored in the value field. Either string or bool.
value The value of the element. The value is either a list of associated keys, text, or a Boolean (true or false).

Keys

A key in the configuration JSON references a specific element of your tutorial setup. The allowed keys include:

Key Type Purpose Format Example
Tutorial list Lists all tutorials. tutorials "tutorials"
Tutorial Lists the steps of a specific tutorial. {TutorialKey} "Tutorial1"
Step text The text associated with a tutorial step {TutorialKey}-{StepKey}-text "Tutorial1-Step2-text"
Step enabled flag Flag indicating whether a tutorial step is enabled. {TutorialKey}-{StepKey}-isEnabled "Tutorial3-Step4-isEnabled"

Tutorial Manager constructs the configuration keys from the key names of the tutorials and steps you enter in the Unity Editor — with the exception of the special key, "tutorials", which is always included in the configuration.

The "tutorials" key element contains a list of all the tutorials in the configuration. Each tutorial, in turn, has an element listing all the steps in that tutorial. The order of the keys in a list determines the order of the tutorials in the configuration and the order of steps within a given tutorial.

The value of elements containing lists are constructed from the associated keys using the format:

"[\"{key_1}\",\"{key_2}\",...,\"{key_n}\"]"

For example, the JSON element with the key, "tutorials" might have the following value for a configuration containing two tutorials: 

"[\"Tutorial1\",\"Tutorial2\"]"

where "Tutorial1" and "Tutorial2" are the names assigned to your tutorials in the Tutorial Editor window.

Note: The format of these stringified lists is designed to be deserialized as an array using a JSON convertor.

Example configuration JSON

{
    "remoteSettings": [{
        "key": "tutorials",
        "type": "string",
        "value": "[\"Tutorial1\",\"Tutorial2\"]"
    }, {
        "key": "Tutorial1",
        "type": "string",
        "value": "[\"Tutorial1-Step1\",\"Tutorial1-Step2\",\"Tutorial1-Step3\"]"
    }, {
        "key": "Tutorial2",
        "type": "string",
        "value": "[\"Tutorial2-Step1\",\"Tutorial2-Step2\",\"Tutorial2-Step3\"]"
    }, {
        "key": "Tutorial1-Step1-text",
        "type": "string",
        "value": "Welcome to my game"
    }, {
        "key": "Tutorial1-Step2-text",
        "type": "string",
        "value": "Here is text for tutorial one, step two"
    }, {
        "key": "Tutorial1-Step1-isEnabled",
        "type": "bool",
        "value": true
    }, {
        "key": "Tutorial1-Step3-isEnabled",
        "type": "bool",
        "value": false
    }]
}
