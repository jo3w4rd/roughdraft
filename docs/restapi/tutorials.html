Tutorial Manager REST API

Manage your tutorial configurations using the REST API.

Version: 2.0.0

/{appid}/tutorials/{environment}

GET

Summary: Download your tutorial configuration from the Tutorial Manager service.

Description: Obtains a copy of the current configuration of all tutorials for a project in the specified environment environment.

Parameters

Name Located in Description Required Schema
appid path The Unity project ID. You can find this id in the Settings section of the Unity Editor Services window or on the Analytics dashboard. Yes string
environment path Must be "Release" or "Development". Yes string
Authorization header The string, Bearer {authorization_tokens.access_token}, where {authorization_tokens.access_token} is the string obtained through the /login API. Yes string

Responses

Code Description Schema
200 OK. The configuration JSON array is returned in the response body. remoteSettings
401 Authorization failed.
404 No existing Tutorial Manager configuration exists for this project or the specified environment name was not recognized (the names are case sensitive).
PUT

Summary: Uploads your tutorial configuration to the Tutorial Manager service.

Description: Replaces the current configuration of all tutorials for a project in the Development environment.

Parameters

Name Located in Description Required Schema
appid path The Unity project ID. You can find this id in the Settings section of the Unity Editor Services window or on the Analytics dashboard. Yes string
environment path Must be Release or Development. Yes string
Authorization header The string, Bearer {authorization_tokens.access_token}, where {authorization_tokens.access_token} is the string obtained through the /login API. Yes string
remoteSettings body The tutorial configuration that replaces the remote version in the development environment. See Configuration structure. Yes remoteSettings

Responses

Code Description
200 OK. The operation succeeded.
400 Bad request. Check that the JSON payload is well formed.
401 Authorization failed.
404 No existing Tutorial Manager configuration exists for this project or the specified environment name was not recognized (the names are case sensitive).
405 Not allowed. You can only put to the Development environment.
422 The configuration is a well-formed JSON object, but contains incorrect or unrecognized elements. See Configuration structure.

/{appid}/tutorials/copy

POST

Summary: Copy the tutorial configuration from the development environment to the Release environment.

Description: Copy.

Parameters

Name Located in Description Required Schema
appid path The Unity project ID. You can find this id in the Settings section of the Unity Editor Services window or on the Analytics dashboard. Yes string
Authorization header The string, Bearer {authorization_tokens.access_token}, where {authorization_tokens.access_token} is the string obtained through the /login API. Yes string
copyParams body Which environment to copy where. Yes copyParams

Responses

Code Description
200 OK. The operation succeeded.
400 Bad request. Check that the JSON payload is well formed.
401 Authorization failed.
404 No existing Tutorial Manager configuration exists for this project or the specified environment name was not recognized (the names are case sensitive).

Models

remoteSettings

Name Type Description Required
remoteSettings [ object ] An array containing the configuration. See Configuration structure. Yes

copyParams

The source and destination for a copy operation.

Name Type Description Required
src string The environment to copy from, either "Development" or "Release". Yes
dest string The environment to copy to, either "Release" or "Development". Yes
